## 1. 핵심 개념 (제일 중요)

이 사이트는 **서버가 없습니다.** GitHub Pages로 정적 호스팅되고, 모든 내용은 저장소의 **`data.js` 파일 하나**에 들어 있습니다.

```
index.html   →  학생이 보는 화면
admin.html   →  원장님이 내용을 입력하는 관리자 콘솔
data.js      →  실제 데이터 (학생·출석·콘텐츠·VIP 등 전부)  ★핵심★
books/       →  업로드한 교재 PDF 파일들
```

**관리자 콘솔(admin.html)에서 내용을 바꾸고 [💾 GitHub에 저장]을 누르면 → `data.js`가 갱신되고 → 1~2분 뒤 학생 화면에 반영됩니다.** 이 한 사이클만 이해하면 됩니다. Cowork(클로드)는 필요 없습니다.

## 2. 집에서 내용 수정하는 3가지 방법

### 방법 A — 관리자 콘솔 (권장, 가장 쉬움)
1. 브라우저에서 `https://docssam1.github.io/hs-report/admin.html` 접속
2. 비밀번호 `01020837265` 입력
3. 탭에서 내용 수정 (출석 체크, 공지 작성, 교재 업로드 등)
4. 하단 **[💾 GitHub에 저장]** 클릭 → 처음 한 번 **GitHub 토큰**을 물어봄(아래 4번 참고) → 저장
5. 1~2분 후 학생 화면 새로고침하면 반영

> 토큰은 그 브라우저에 저장돼서 다음부터는 안 물어봅니다.

### 방법 B — GitHub 웹에서 직접 편집
- `data.js`를 GitHub에서 열고 ✏️(연필) 아이콘 → 수정 → Commit.
- "응원 한마디" 같이 관리자 UI에 없는 항목을 고칠 때 사용 (아래 6번 참고).

### 방법 C — 내 컴퓨터에서 편집
- 저장소를 `git clone` → 파일 수정 → `git add . && git commit && git push`.
- 개발 환경이 익숙하면 이 방법.

## 4. GitHub 토큰(PAT) — 저장/업로드에 필요

관리자 콘솔의 "저장", "PDF 업로드"는 GitHub에 직접 쓰기 때문에 **개인 토큰**이 필요합니다.

발급 (한 번만):
1. GitHub 로그인 → 오른쪽 위 프로필 → **Settings**
2. 맨 아래 **Developer settings** → **Personal access tokens**
3. **Fine-grained tokens → Generate new token**
   - Repository access: `docssam1/hs-report` 선택
   - Permissions → **Contents: Read and write**
   - (또는 Classic token + `repo` 권한)
4. 생성된 토큰(`github_pat_...`)을 복사 → 관리자 콘솔이 물어볼 때 붙여넣기

> 토큰은 사이트가 아니라 **그 브라우저(내 PC)에만** 저장됩니다. 공용 PC에서는 저장하지 마세요. 잃어버리면 새로 발급하면 됩니다.

## 6. "응원 한마디" 수정 (이번 주 화면 멘트)

이번 주 화면 맨 위 브리핑의 **DOCSSAM 한마디**는 `data.js`의 다음 두 곳에서 옵니다. (현재는 관리자 UI가 없어 **방법 B(GitHub 직접 편집)**로 고칩니다.)

```js
"vip": {
  ...
  "coach": [            // 학생용 응원 (낮·저녁 등 학생이 볼 시간대)
    "오늘 푼 한 문제가 ...",
    "실수해도 괜찮아 ..."
  ],
  "coachParent": [      // 학부모용 (학교 시간·밤 시간대 자동 전환)
    "아이가 영상 보고 다시 푼 문제를 ...",
    "결과보다 노력을 칭찬해 주세요 ..."
  ]
}
```

- 문장을 추가·수정·삭제하면 됩니다. 날짜에 따라 매일 하나씩 자동 순환합니다.
- **시간대 규칙**: 평일 08~16시·밤 22~06시 = 학부모 멘트, 그 외 = 학생 멘트로 자동 전환.
- D-day(시험까지 남은 날)·국면 문구는 코드가 자동 계산하므로 손댈 필요 없습니다.

## 8. data.js 데이터 구조 요약

```
meta            : 학원명·제목·현재주차(currentWeekId)·시험일(examDate)·좌표(lat,lng)
nodes[]         : 로드맵 노드(주차/특강/구분선/목표)
content{}       : 주차별 { notice(학습내용), homework(과제), textbooks[] }
students[]      : 학생 이름 목록
attendance{}    : 학생별 출석한 주차 id 목록 (권한)
specialStudents : 방학특강 수강생
archiveFolders[] / archiveAccess{} : 자료실 폴더 + 폴더별 공개 범위(["*"]=전체)
books[]         : 교재 { folder,title,pdf,video,cover,date,links[] }
vip.sessions[]  : 프리미엄 설명회 영상 { title,url,thumb,date }
vip.courses[]   : 신규 강좌 공지 { title,body,link,date }
vip.clinic      : 결석·클리닉 { notice, url(네이버 폼) }
vip.columns[]   : DOCSSAM 칼럼 { title,category,date,html }
vip.magazine[]  : 교육 뉴스 { category,title,desc,source,sourceUrl,image,date }
vip.coach[] / vip.coachParent[] : 브리핑 응원 한마디
pins{}          : 학생별 진단 결과 잠금번호(전화 뒷 4자리) — "이름":"1234"
reports{}       : 학생별 진단 결과 목록 — "이름":[{ title,date,score,total,grade,areas[],comment }]
```

> **진단 결과 잠금**: `pins`에 뒷자리가 있으면 학생이 그 번호를 입력해야 자기 진단 결과가 열립니다. 뒷자리를 비우면 잠금 없이 바로 열립니다. (데이터는 학생별로 분리되어 다른 학생 것은 안 보임)

> `data.js`는 반드시 맨 위 `window.GFIELD_DATA = {` 로 시작하고 맨 끝 `};` 로 끝나야 합니다. 중간은 정상적인 JSON 문법(쌍따옴표, 쉼표)을 지켜야 합니다. 직접 편집 시 쉼표 빠짐을 주의하세요.

## 9. 자주 하는 작업 — 빠른 체크리스트

- **이번 주 영상·자료 열기** → ① 출석·권한에서 학생 칸 클릭 → 저장
- **이번 주 안내문 올리기** → ③ 콘텐츠에서 해당 주차 학습내용/과제 붙여넣기 → 저장
- **교재 PDF 올리기** → ⑤ 자료실 → 교재 추가 → PDF 선택(자동 업로드) → 저장
- **설명회 영상 추가** → ⑥ VIP → 설명회에 유튜브 링크 추가 → 저장
- **클리닉 신청 연결** → ⑥ VIP → 클리닉 칸에 네이버 폼 주소 → 저장
- **칼럼/뉴스 쓰기** → ⑦ 교육 매거진 → 작성/AI 생성 → 저장
- **진단 결과·코멘트 입력** → ⑧ 진단 결과 → 학생 선택 → 뒷 4자리(잠금) + 회차·점수·영역·코멘트 → 저장

## 10. 문제 해결

| 증상 | 해결 |
|---|---|
| 저장이 안 됨 / "토큰이 유효하지 않습니다" | GitHub 토큰 재발급 후 다시 입력 (Contents 쓰기 권한 확인) |
| 바꿨는데 학생 화면에 안 보임 | 1~2분 대기 + 새로고침(또는 시크릿창). GitHub Pages 반영 지연 |
| 날씨가 안 뜸 | "이번 주" 화면에서 확인(다른 탭엔 없음). 네트워크 일시 오류면 잠시 후 자동 재시도 |
| AI 생성 실패 | Gemini 키 확인 / ⑦의 모델명을 다른 값으로 변경 |
| 캐릭터 성별 바꾸고 싶음 | 현재 이름 기반 자동. 학생별 지정이 필요하면 개발 수정 필요 |
| 학생이 진단 결과가 안 열림 | ⑧에서 그 학생 **뒷 4자리**를 정확히 넣었는지 확인. 학생은 전화번호 뒷 4자리를 입력해야 열림 |
